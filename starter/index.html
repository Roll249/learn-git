Bài 0: Từ folder trên máy lên GitHub

Đây là bài bắt buộc cho người mới. Làm đúng 5 bước dưới đây để biến folder local thành repository trên GitHub.

0. Cài Git trước khi bắt đầu

Nếu máy bạn chưa có Git, cài theo hệ điều hành bên dưới rồi kiểm tra lại bằng git --version.

Windows

  1. Tải Git tại: git-scm.com/download/win
  2. Mở file cài đặt và để mặc định các bước Recommended.
  3. Mở Git Bash hoặc PowerShell, chạy:
git --version

macOS

  1. Cách nhanh: mở Terminal và chạy:
git --version
  1. Nếu máy chưa có Git, macOS sẽ gợi ý cài Xcode Command Line Tools, chọn Install.
  2. Hoặc cài bằng Homebrew:
brew install git

Linux

Chọn lệnh theo distro:

sudo apt update && sudo apt install -y git sudo dnf install -y git sudo pacman -S git

Kiểm tra sau cài đặt: git --version

Windows chi tiết (từng click)

  1. Tải bộ cài Git tại git-scm.com/download/win.
  2. Mở installer và giữ các lựa chọn Recommended (mặc định), bấm Next liên tục.
  3. Đến bước PATH, chọn tùy chọn cho phép dùng Git từ command line.
  4. Hoàn tất cài đặt, mở Git Bash và chạy git --version.

Bước 1 — Mở đúng folder local

Trong terminal, chuyển tới thư mục dự án bạn muốn quản lý bằng Git.

cd /path/to/your-project ls

Bước 2 — Khởi tạo Git và commit đầu tiên

Tạo repository local và lưu snapshot đầu tiên.

git init git add . git commit -m "chore: initial commit" git branch -M main

Bước 3 — Tạo repo mới trên GitHub

Vào GitHub và chọn tạo repo mới. Không tick tạo README/.gitignore tự động.

GitHub · New repository
  • Repository name: my-project
  • Visibility: Public hoặc Private
  • Không chọn: Add a README / .gitignore / license

Bước 4 — Kết nối local với remote

Copy URL repository vừa tạo và gắn vào local.

git remote add origin <repo-url> git remote -v

Bước 5 — Push lần đầu

Đẩy code lên GitHub và thiết lập upstream để các lần sau đơn giản hơn.

git push -u origin main
GitHub · Repository files

Nếu thấy danh sách file xuất hiện là hoàn tất 🎉

Lỗi thường gặp và cách xử lý nhanh

  • remote origin already exists: chạy git remote remove origin rồi add lại.
  • src refspec main does not match any: chưa có commit đầu, quay lại bước 2.
  • permission denied (publickey): cấu hình SSH key hoặc dùng HTTPS URL.
