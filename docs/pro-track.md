# Professional Track Add-on

## Quy chuẩn commit

- Sử dụng dạng: `type(scope): summary`.
- Một commit = một ý thay đổi chính.

## Quy chuẩn Pull Request

- PR nhỏ, mô tả rõ mục tiêu.
- Có checklist test/manual verify.
- Nêu rủi ro và hướng rollback.

## Code Review Etiquette

- Comment vào hành vi code, không phán xét cá nhân.
- Ưu tiên câu hỏi làm rõ trước khi yêu cầu sửa.
- Merge khi đạt đồng thuận và passing checks.
